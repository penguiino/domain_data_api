What is solid

FastAPI structure with versioned routing (/v1).

Deterministic, read-only endpoints backed by static JSON.

Basic rate limiting (SlowAPI).

API-key gate exists conceptually.

What is missing / insufficient

API key management

Keys hardcoded in source.

No key rotation, revocation, expiry, per-key quotas, or attribution.

Keys not stored securely (env/secret manager).

Authentication enforcement

require_api_key is not applied globally or per-route via dependencies.

Any endpoint without the dependency is effectively public.

Production configuration

No environment separation (dev/staging/prod).

No settings via environment variables.

No secret management.

Observability

No structured logging.

No request/response metrics.

No health/readiness endpoints.

Error handling

No global exception handling beyond rate limits.

File I/O errors can leak stack traces.

Rate limiting model

IP-based only; not per API key.

No burst control or differentiated plans.

Security hardening

No CORS policy defined.

No HTTPS enforcement (deployment-side but undocumented).

No request size limits.

No dependency vulnerability pinning beyond requirements.txt.

Data loading strategy

JSON loaded from local filesystem; unsuitable for multi-instance deployments without shared storage.

Cache invalidation via mtime + lru_cache is fragile under concurrency.

Testing

No automated tests (unit/integration).

No contract tests for API stability.

Operational readiness

No Dockerfile / deployment artifacts.

No CI pipeline.

No SLA/version deprecation policy.

Where to go next (minimal path to production)

Enforce API keys via FastAPI dependencies on all routes; move keys to env/DB.

Implement a real key system: hashed keys, metadata, per-key rate limits.

Switch rate limiting to API-key–based.

Add structured logging, /health, and basic metrics.

Add global exception handlers with sanitized errors.

Externalize configuration (env vars); add Dockerfile.

Add tests for all endpoints.

Document auth, limits, and versioning guarantees.
