🧠 Core idea

A delayed-feedback forecasting app.
Users make probabilistic predictions about real-world prompts, then receive feedback weeks or months later when outcomes resolve. The design emphasizes commitment, reflection, and calibration, not speed or feeds.

🧱 Tech stack

Flutter (Android-first)

Riverpod 2.x for state management

GoRouter for navigation + auth gating

Firebase

Authentication (email/password)

Firestore (users, prompts, forecasts)

MVVM architecture

📁 Architecture overview
lib/

👤 User flows
1️⃣ Authentication

Email/password signup

Username uniqueness enforced

Login / logout

Auth state synced to router

Automatic redirect:

Logged out → /login

Logged in → /feed

2️⃣ Feed (Prompt discovery)

Streamed from Firestore (prompts collection)

Sorted by createdAt

Each prompt has:

text

status: open or resolved

outcome (if resolved)

Tap behavior:

Open + no forecast → Forecast submission

Open + already forecasted → Forecast detail

Resolved → Forecast detail (with or without user forecast)

3️⃣ Forecast submission

Probability slider (0–100%)

Rationale text input

Immutable after submission

Forecast stored in forecasts collection:

userId

promptId

probability

rationale

timestamp

4️⃣ Forecast detail

Handles multiple cases:

User has forecast

User didn’t forecast

Prompt resolved or unresolved

Displays:

Prompt text

Forecast card (if exists)

Live status (open / resolved)

Outcome (when resolved)

Score + XP (when resolved & predicted)

Comments section

5️⃣ Profile

User information

Forecast history

XP

Count of forecasts made

Tap any forecast → Forecast detail page

6️⃣ Resolution & scoring (admin / future logic)

Admin can resolve prompts (true / false)

Resolution triggers:

Score calculation

XP assignment

Designed to be extended later (AI or API-based resolution)

🧭 Navigation rules (important)

Router listens to Firebase auth stream

Auth-aware redirects prevent invalid states

/forecast_detail is data-driven, not path-driven:

Uses state.extra (UserForecast or prompt map)

Prevents:

Editing forecasts

Accessing protected pages while logged out

🧩 Known constraints (intentional)

No editing forecasts after submission

No “instant feedback”

No infinite scrolling or engagement loops

Minimal UI, dark theme

Android-only for now
